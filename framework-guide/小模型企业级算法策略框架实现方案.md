# 小模型算法策略 — 企业级方案（与实现对齐）

> 与《[小模型通道推理整体实现技术说明](./小模型通道推理整体实现技术说明.md)》配合阅读。  
> **落地说明（简表）**：`enterprise-level_transformation_docs/小模型算法策略实现说明.md`

---

## 1. 目标

- 算法按场景**配置化**（`algor_type` + `small_model_algorithms.yaml` + API 覆盖）。  
- **调用扁平**：推理引擎只调度 `strategy/` 下少量策略类 + `_yolo_utils`。  
- **可运维**：权重路径、阈值、ROI、类过滤、复杂时空参数均可配置。

---

## 2. 策略分层（企业级分类）

| 层级 | 策略类 | 说明 |
|------|--------|------|
| L1 常规目标 | `ObjectDetectionStrategy` | 静态/属性类目标，多场景共用 YOLO 检测范式 |
| L2 常规行为 | `RegularBehaviorDetectionStrategy` | 行为类，多为自训 YOLO；实现路径与 L1 一致，文件分离便于治理 |
| L3 复杂时空 | `ComplexBehaviorDetectionStrategy` | YOLO + `dwell` / `line_cross` / `zone_intrusion` |

**扩展**：分割、姿态、跟踪、整图分类等 → 新增 `strategy/xxx.py` 并在 `inference_engine._STRATEGY_CLASSES` 注册一行。

**兼容**：YAML 中旧名 `CallingStrategy` → 引擎内映射为 `RegularBehaviorDetectionStrategy`（同一单例）。

---

## 3. 目录与职责

| 路径 | 职责 |
|------|------|
| `app/small_models/strategy/object_detection.py` | L1 |
| `app/small_models/strategy/regular_behavior_detection.py` | L2 |
| `app/small_models/strategy/complex_behavior_detection.py` | L3 |
| `app/small_models/strategy/_yolo_utils.py` | YOLO 加载与几何小工具 |
| `app/small_models/strategy/base.py` | `Detection` / `StrategyResult` / `SmallModelStrategy` |
| `app/small_models/inference_engine.py` | 合并配置、选策略、证据、回调 |
| `app/small_models/algorithm_registry.py` | `AlgorithmConfig` 与 YAML 加载 |
| `configs/small_model_algorithms.yaml` | `algor_type` 默认项 |

---

## 4. 配置优先级

1. API / `extra_params` 非空字段  
2. `small_model_algorithms.yaml`  
3. `SmallModelRegistry` 默认 `weights_path`（若存在）

---

## 5. 非功能要求（摘要）

- 单通道解码/推理线程模型不变；策略**无**长阻塞。  
- 单帧推理异常记录日志，不拖死线程。  
- 复杂行为状态键包含 `channel_id` + `algor_type`，避免串通道。

---

## 6. 变更说明

早期方案中的 `perception/`、`pipeline/`、`reasoning/`、`strategy_registry` 等分层**已移除**，以减轻调用深度；若文档其他处仍提及，以本文与 `enterprise-level_transformation_docs` 为准。
