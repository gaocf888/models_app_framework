# 数据持久化与容器部署说明

> 目标：梳理本项目在 Docker / K8s 等容器环境下**需要持久化的数据**与**推荐挂载目录**，避免容器销毁导致业务状态或训练成果丢失。

---

## 1. 总览：需要持久化的模块

- **会话与上下文（Redis）**
  - 模块：`ConversationStore` + `ConversationManager`
  - 作用：存储用户会话历史、摘要等上下文。
  - 持久化建议：**不在本服务容器内持久化**，而是使用**独立 Redis 实例**（单独容器或托管服务），通过 `REDIS_URL` 连接。

- **RAG 检索库（ES/EasySearch，默认）+ FAISS（可选）**
  - 模块：`ElasticsearchVectorStore` / `FaissVectorStore` + `RAGService` + `RAGIngestionService`
  - 默认数据形态：ES/EasySearch 中的向量 + 全文 + 元数据索引（chunks/docs/jobs）。
  - 持久化建议（默认）：
    - ES/EasySearch 作为独立容器或托管服务部署，数据目录挂载持久卷；
    - 应用容器仅通过 `RAG_ES_HOSTS`、`RAG_ES_USERNAME`、`RAG_ES_PASSWORD` 等环境变量连接。
  - 参考部署包：`rag_db-deploy/`（含 Docker Compose、配置模板、初始化脚本、项目侧 env 模板）。
  - FAISS（可选）仍可用于开发/离线场景，需持久化 `RAG_FAISS_INDEX_DIR`。

- **业务数据库（MySQL 等）**
  - 模块：`DatabaseConfig` / NL2SQL 子系统（`SchemaMetadataService`、`SQLExecutor` 等）。
  - 数据：业务库表（`aishare` 等）。
  - 持久化建议：数据库通常作为**独立服务**部署（单独容器或托管 MySQL），由数据库自身的数据目录（如 `/var/lib/mysql`）持久化；本项目容器仅通过 `DB_URL` 连接。

- **训练产物与模型权重**
  - 模块：
    - 大模型训练：`app/train/llm_training.py`、`TrainingOrchestrator`、`LLaMAFactoryAdapter`；
    - 小模型训练：`SmallModelTrainingService`。
  - 数据：
    - 大模型微调输出的权重（checkpoint、LoRA 权重等）；
    - 小模型训练生成的权重文件；
    - 训练日志与 TensorBoard 日志目录。
  - 持久化建议：统一配置训练输出根目录（如 `/data/models`、`/data/train_logs`），并映射为持久卷。

- **日志与监控**
  - 模块：
    - 日志：`LoggingConfig` / `LoggingManager`；
    - 指标：`/metrics`（Prometheus 抓取）。
  - 建议：
    - 应用日志推荐**输出到 stdout/stderr**，由宿主机 / 日志系统（如 Loki）收集，无需在容器内额外挂盘；
    - 如开启文件日志（`LoggingConfig.log_file`），则对应日志目录建议挂载持久卷；
    - Prometheus / Grafana / Loki 等自身的数据目录由各自容器的运维配置持久化，本服务无需额外处理。

## 2. 模块级持久化清单与 Docker 挂载建议

### 2.1 会话与上下文（Redis）

- **是否需要本仓库容器挂盘**：否。
- **部署建议**：
  - 使用外置 Redis（独立容器或托管服务），本服务通过 `REDIS_URL` 连接；
  - 在生产环境为 Redis 开启 AOF/RDB 等持久化机制，由 Redis 容器自身的数据目录挂盘。

### 2.2 RAG 检索库（ES/EasySearch，默认）

- **配置项**：
  - `RAG_VECTOR_STORE_TYPE=es`（默认）；
  - `RAG_ES_HOSTS`、`RAG_ES_USERNAME`、`RAG_ES_PASSWORD`、`RAG_ES_API_KEY`；
  - `RAG_ES_INDEX_*`、`RAG_ES_DOCS_INDEX_*`、`RAG_ES_JOBS_INDEX_*`。
- **持久化对象**：
  - EasySearch/Elasticsearch 容器数据目录（例如 `/usr/share/elasticsearch/data`）；
  - 索引/别名配置由应用自动 migration 或初始化脚本管理。
- **容器部署建议**：
  - 优先使用仓库内部署包：`rag_db-deploy/README.md`；
  - 通过 `rag_db-deploy/docker-compose.easysearch.yml` 启动数据库容器；
  - 应用容器通过 `rag_db-deploy/project-env/rag-es.env.example` 对接连接参数。

### 2.3 RAG 向量库（FAISS，可选）

- **适用场景**：开发、离线单机、无 ES 环境。
- **配置项**：
  - `RAG_VECTOR_STORE_TYPE=faiss`；
  - `RAG_FAISS_INDEX_DIR`（默认 `./data/faiss`）。
- **需要持久化的文件**：
  - `<RAG_FAISS_INDEX_DIR>/faiss.index`
  - `<RAG_FAISS_INDEX_DIR>/faiss_meta.json`
- **容器部署示例**：

```bash
docker run -d \
  --name rag-service \
  -e RAG_VECTOR_STORE_TYPE=faiss \
  -e RAG_FAISS_INDEX_DIR=/data/rag_index \
  -v /opt/rag_index:/data/rag_index \
  your-image:tag
```

- **说明**：
  - 只要映射好该目录，容器重启或重新部署后，索引可从 `faiss.index` / `faiss_meta.json` 自动加载。

### 2.4 业务数据库（MySQL 等）

- **是否在本容器内持久化**：否。
- **做法**：
  - 数据库以独立方式部署，保证其数据目录挂盘（如 Docker Compose / K8s StatefulSet 中配置 volume）；
  - 本项目容器通过环境变量配置数据库连接：
    - `DB_URL` 或 `DB_HOST` / `DB_USER` / `DB_PASSWORD` / `DB_NAME`。

### 2.5 训练产物与模型权重

- **配置建议**（示例，可按业务调整）：
  - 大模型训练输出根目录：`/data/llm_models`
  - 小模型训练输出根目录：`/data/small_models`
  - TensorBoard 日志：`/data/tensorboard`

- **容器挂载示例**：

```bash
docker run -d \
  --name llm-dajia \
  -e LLM_TRAIN_OUTPUT_DIR=/data/llm_models \
  -e SMALL_MODEL_TRAIN_OUTPUT_DIR=/data/small_models \
  -e TENSORBOARD_LOG_DIR=/data/tensorboard \
  -v /opt/llm_models:/data/llm_models \
  -v /opt/small_models:/data/small_models \
  -v /opt/tensorboard:/data/tensorboard \
  your-image:tag
```

- **说明**：
  - 训练脚本应统一读取上述环境变量作为输出目录；
  - 运维上可将这些目录纳入备份策略。

### 2.6 日志与监控

- **应用日志**：
  - 推荐：日志写到 stdout/stderr，由 Docker / K8s / Loki 收集；
  - 如需落地到文件（`LoggingConfig.log_file`），建议：
    - 日志目录如 `/data/logs/app`；
    - 容器挂载：`-v /opt/app_logs:/data/logs/app`。

- **监控数据（Prometheus / Grafana / Loki）**：
  - 这些组件通常以独立部署方式存在，各自的持久化不由本容器负责；
  - 本服务只需暴露 `/metrics`，及输出结构化日志。

## 3. Docker / K8s 部署时的通用建议

1. **配置与密钥分离**  
  - 使用环境变量（如 `DB_URL`、`REDIS_URL`、`RAG_ES_HOSTS`、`RAG_FAISS_INDEX_DIR` 等）配置外部依赖；
   - 不在镜像内写死账号密码，生产环境可结合 K8s Secret / 外部配置中心。

2. **最小必要挂载**  
   - 对本项目容器本身，通常只需要挂载：
     - 若使用 FAISS：RAG 索引目录（如 `/data/rag_index`）；
     - 若使用 ES/EasySearch：数据库容器数据卷（应用容器无需挂载 RAG 索引目录）；
     - 训练输出与日志目录（如 `/data/llm_models`、`/data/small_models`、`/data/tensorboard`、`/data/logs/app`，按需选择）。

3. **多环境隔离**  
   - 不同环境（dev / test / prod）使用不同的存储路径与数据库/Redis/向量库实例，避免数据串用；
   - 可以通过 `APP_ENV` 与不同的挂载路径组合管理。

4. **备份与恢复**  
   - 对于关键数据（业务数据库、训练产物、重要 RAG 知识库），建议纳入备份策略；
   - RAG 索引若可通过离线摄入任务重建，则可把备份重点放在“原始知识源”和“摄入任务定义”，索引目录采取简单冷备即可。
