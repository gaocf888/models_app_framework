## 1. 定位与范围

**是什么**：大模型在线应用平台——FastAPI 暴露推理、客服、综合分析、检修报告结构化提取、NL2SQL、RAG 运维等能力；**LangChain / LangGraph** 编排；**自研 RAG**（多路召回 + RRF + CrossEncoder）与 **NL2SQL**、会话、观测为基座。

**覆盖**：`app/main.py` 挂载的健康检查、`/llm`、`/chatbot`、`/analysis`、`/inspection-extract`、`/nl2sql`、`/rag` 及公共基础设施。

**不覆盖**：`/small-model`、`/dajia` 及 `app/train/`、`app/small_models/` 等小模型与训练管理面（见专项文档）。

---

## 2. 逻辑分层（一览）

| 层级 | 职责 |
|------|------|
| 接入 | FastAPI、OpenAPI、CORS、`/metrics`、异常映射 |
| 安全 | Service API Key（Bearer），`app/auth/dependencies.py` |
| 应用服务 | `*_service.py` 封装业务，注入 RAG/会话/链/图 |
| 编排 | LangChain 链、LangGraph（客服/分析）、Agentic RAG |
| 领域 | `app/rag/`、`app/nl2sql/`、`app/graph/`（可选 GraphRAG） |
| 基础设施 | 统一配置、日志、Prometheus、会话存储 |

---

## 3. 外部依赖（在线）

**vLLM**（OpenAI 兼容 HTTP）、**ES / EasySearch**（向量+全文，可选 FAISS）、**Redis**（会话、trace、checkpoint 等）、**业务数据库**（NL2SQL 目标库）、**Neo4j**（可选 GraphRAG）、**MinerU**（可选 PDF 解析）。

调用方经 HTTPS 访问，请求头携带 Bearer 服务密钥（对应环境变量 `SERVICE_API_KEY` 或 `SERVICE_API_KEYS`）。

---

## 4. 基座与业务（架构一句话）

**RAG**：`RAGService` — 语义 + 关键词（+ 可选元数据）并行 → RRF → CrossEncoder → top_k；`HybridRAGService` 叠图路由；`AgenticRAGService` 子查询与预算；嵌入 sentence-transformers；存储 `VectorStoreProvider`（ES/FAISS）；摄入异步 Job；**主检索非 LangChain RAG 链**。

**NL2SQL**：LangChain 多步链 + `NL2SQLService`；只读校验 + 受控执行；多入口复用。

**LLM**：`VLLMHttpClient`（httpx）与 LangChain `ChatOpenAI` 双轨回退；`LLMConfig` 路由。

**会话**：`ConversationManager` + Redis/内存；Redis 下独立 asyncio 线程隔离 I/O。

| 业务面 | 编排要点 |
|--------|----------|
| `/chatbot` | 入口先做图片预处理（缩放/压缩/本地落盘）再进 LangGraph（意图 → RAG/NL2SQL → 流式）；可 Legacy；SSE `meta` 含 `processed_image_urls`，会话查询返回 `image_urls` |
| `/analysis` | payload / nl2sql：**两套 LangGraph**（`AnalysisGraphRunner`）；**看图诊断**：`run-img-diag` + `img-diag/upload`，**`AnalysisImgDiagGraphRunner`** 并行三臂 + 合成（非第三套 StateGraph）；trace 旁路与 **`by_data_mode` 含 `img_diag`**；专项文档见 **`企业级综合分析-看图诊断实现和使用说明.md`** |
| `/inspection-extract` | 两段式（upload -> run）：文档上 MinIO 后通过 URL 解析；docx/pdf 自动路由、扫描 PDF 可走 MinerU；LLM 三阶段抽取 + 规则后处理（阈值绑定/缺陷归一） |
| `/llm` | 轻编排；可选 Agentic + RAG；共享基座 |
| `/rag` | REST + 异步摄入；管理写与在线只读分离；不经 LangGraph |
| `/nl2sql` | HTTP → `NL2SQLService` → LangChain 链 |

## 5. HTTP 门面（速查）

| 前缀 | 能力 |
|------|------|
| `/health`、`/api/health` | 健康检查 |
| `/metrics` | Prometheus |
| `/llm` | 通用推理（可选 RAG / Agentic） |

| `/rag` | 知识库运维与摄入 |

业务路由需 **Bearer Service API Key**（未配置密钥时 503）。

## 6. 技术栈与数据面（压缩）

- **运行时**：Python 3.10+，FastAPI，Pydantic v2，httpx。  
- **AI 编排**：LangChain、LangGraph；可选 LangSmith。  
- **观测**：Prometheus、`/metrics`；结构化日志。  
- **部署**：容器化，`app/core/config.py` + 环境变量；见 `app/app-deploy/README.md`。

| 数据 | 存储 | 用途 |
|------|------|------|
| 向量/全文 chunk | ES（或 FAISS） | RAG 检索 |
| 文档/任务元数据 | ES 索引 | 运维与摄入 |
| 会话 | Redis / 内存 | 多轮 |
| trace | Redis / 内存（可选 ES） | 分析排障 |
| 图 | Neo4j（可选） | GraphRAG |
| 查询结果 | MySQL 等 | NL2SQL |

---


## 7. 小结

平台以 **FastAPI + Service API Key** 统一对外；智能能力建立在 **vLLM 兼容推理**、**ES 混合 RAG（自研多路+RRF+重排）**、可选 **GraphRAG** 与 **LangChain/LangGraph 编排**之上；**NL2SQL**、**会话**、**指标日志**横切全链路；配置与部署通过 **环境变量 + 容器栈** 在多环境复制与审计。
