# 项目概述与目标（大小模型统一应用基座）

## 1. 项目定位

本项目旨在构建一套**面向大小模型（LLM/多模态 + YOLO 等小模型）应用开发的通用技术基座**，抽象出与具体业务无关的通用能力，包括：

- **大模型推理与训练/微调封装**（基于 vLLM + LangChain，支持多模型、多模态）。
- **小模型推理检测与训练封装**（参考 `ym_new` 工程的视频流通道方案，强调多线程与线程安全，训练同时支持代码方式与 TensorBoard 可视化）。
- **RAG、上下文与会话管理**（支持传统 RAG + Agentic RAG，Redis 持久化上下文）。
- **智能客服 / ChatBot 能力**（RAG 可配置、流式响应、多用户会话状态管理）。
- **综合分析能力**（Agentic RAG + LangChain，支持图像/视频/GPS/传感器等多源数据综合分析）。
- **NL2SQL 子系统**（基于现有《NL2SQL系统概要设计》进行组件化封装）。
- **统一的 API 接入层**（FastAPI 异步接口，兼容大模型高并发与小模型多线程管道）。
- **全局日志体系与可观测性**（统一日志规范 + Prometheus/Grafana/Loki 监控链路）。

该基座不绑定具体业务场景，可在其之上快速构建智能客服、风控分析、视频监控、运营分析等上层应用。

## 2. 目标

- **统一性**：为所有大小模型相关业务提供统一的架构、组件与调用规范，避免各项目各自为政。
- **高并发 & 线程安全**：
  - 大模型：请求级高并发、任务队列 + Worker Pool、LangChain 实例与 vLLM 调用链的线程安全与性能保证。
  - 小模型：严格区分「视频解码 → 任务队列 → 推理检测」的通道式架构，确保多路视频、多任务下线程安全。
- **高可用 & 可观测**：涵盖日志、指标、追踪与告警，支撑生产环境运维与问题追溯。
- **可扩展性**：便于新增模型类型（新 LLM、多模态、小模型）、新增能力模块（如代码生成、Agent 工作流等）。
- **可配置性**：RAG/上下文/会话、提示词版本、A/B 测试等可配置，支持在不改代码的情况下切换策略。

## 3. 技术栈

- **语言与运行环境**
  - Python 3.10+
  - 异步 + 多线程混合模型（FastAPI async + 小模型线程池 + 后台任务执行器）

- **大模型与 LangChain 生态**
  - vLLM 作为大语言模型 / 多模态模型的主要推理引擎（HTTP/gRPC 服务）。
  - LangChain/LangGraph 作为链路编排与 Agentic RAG 框架：
    - LLM 调用抽象、聊天历史与内存模块、RAG 链路、Agent 工具调用等。
    - 集成 LangSmith 用于链路监控、提示词迭代与 A/B 测试。

- **小模型与推理框架**
  - PyTorch / ONNX Runtime / TensorRT 等作为小模型推理后端。
  - 视频解码：OpenCV / FFmpeg 等。
  - 线程模型参考 `ym_new` 工程：通道管理 + 解码线程 + 推理线程 + 有界消息队列。

- **数据与缓存**
  - Redis：会话与上下文持久化、分布式锁、简单队列。
  - 业务数据库（MySQL/PostgreSQL 等）：由上层业务使用；NL2SQL 通过统一数据访问适配器访问。
  - 向量数据库：FAISS / Milvus / pgvector / ElasticSearch dense vector。

- **Web/API 层**
  - FastAPI：异步 REST/WebSocket 接口；与 Uvicorn/Gunicorn 配合实现高并发。

- **日志与监控**
  - 日志：标准 `logging` + 统一 JSON 格式封装（便于 Loki 收集）。
  - 指标：Prometheus（请求 QPS、延迟、错误率、模型调用耗时、队列长度等）。
  - 可视化：Grafana。
  - 日志聚合：Loki。
