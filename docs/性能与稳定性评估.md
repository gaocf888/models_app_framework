# 性能与稳定性评估（骨架）

> 说明：本文件作为性能与稳定性专项（P12/P13）的评估与记录骨架，当前仅提供结构与示例，实际压测结果与调优结论需在目标环境完成后补充。

## 1. 目标接口与场景

- `/llm/infer`：通用大模型推理接口。
- `/chatbot/chat`：智能客服兼容非流式接口。
- `/chatbot/chat/stream`：智能客服主用 SSE 流式接口。
- `/analysis/run-with-payload`、`/analysis/run-with-nl2sql`：综合分析企业版 V2 接口。
- `/nl2sql/query`：NL2SQL 查询接口。
- `/small-model/channel/*`：小模型通道管理与推理相关接口。

## 2. 压测工具与方法（建议）

- 推荐：
  - `locust`（Python 生态友好，便于脚本化管理多场景）。
- 备选：
  - `k6`（使用 JavaScript 定义场景）。
  - `JMeter`（适合已有经验的团队）。

当前代码仓中提供了基础的 Python 压测骨架脚本：

- `benchmarks/bench_llm_infer.py`：针对 `/llm/infer` 的简单并发压测骨架；
- `benchmarks/bench_chatbot.py`：针对 `/chatbot/chat`（可扩展为流式场景）；
- `benchmarks/bench_nl2sql.py`：针对 `/nl2sql/query`。

这些脚本使用 `httpx.AsyncClient` 并发发送请求，计算 QPS 与 P95/P99 延迟，便于在开发/测试环境快速获得粗略指标。

## 3. 指标维度（建议）

- 性能指标：
  - QPS/TPS；
  - 平均延迟、P95、P99；
  - 错误率（HTTP 5xx / 业务错误）。
- 资源使用：
  - CPU 使用率；
  - 内存占用；
  - GPU 利用率（如有）；
  - 网络带宽。
- 稳定性：
  - 长时间压测下的错误趋势；
  - 内存泄漏或资源未释放情况。
