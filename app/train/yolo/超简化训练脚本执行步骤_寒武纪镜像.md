# 寒武纪镜像：超简化 YOLO 训练执行步骤

本文档仅针对脚本：`app/train/yolo/yolo_train_super_simple.py`。  
该脚本已写死以下关键配置：

- 预训练权重：`app/train/yolo/weights/pretrained/yolov8s.pt`
- 数据集配置：`app/train/yolo/datasets/processed/dajia/dataset.yaml`
- 训练参数：`imgsz=640, epochs=150, batch=30, workers=16, device=0,1,2,3,4,5` 等

## 重要说明

当前版本训练目录（`app/train/yolo`）中的 `ultralytics/` 为适配寒武纪的版本。  
脚本执行 `from ultralytics import YOLO` 时，优先使用当前项目内该版本，请确保完整项目目录已同步到服务器。

---

## 怎么执行（按顺序做）

1. **把完整项目放到服务器**

   示例路径：`/data/projects/models_app_framework`

2. **确认预训练权重文件已就位**

   必须存在：

   - `app/train/yolo/weights/pretrained/yolov8s.pt`

3. **确认数据集目录与配置**

   必须存在：

   - `app/train/yolo/datasets/processed/dajia/dataset.yaml`
   - `app/train/yolo/datasets/processed/dajia/images/train`
   - `app/train/yolo/datasets/processed/dajia/images/val`
   - 对应标签目录（通常为 `labels/train`、`labels/val`）

   `dataset.yaml` 至少包含：

   ```yaml
   path: app/train/yolo/datasets/processed/dajia
   train: images/train
   val: images/val
   names:
     0: person-fight-
     1: Violence
     2: abnormal-behavior
   ```

4. **进入寒武纪容器执行训练**

   在项目根目录运行（`-w` 已指定）：

   ```bash
   docker run --rm -it \
     --privileged \
     -v /opt/deeprev/train/models_app_framework_train:/workspace/models_app_framework_train \
     -w /workspace/models_app_framework_train \
     ultralytics_mlu:v1.1 \
     bash -lc 'PYTHONPATH=. python app/train/yolo/yolo_train_super_simple.py'
   ```

   寒武纪设备透传参数（如 `--device`、runtime 等）按你们平台规范追加到 `docker run`。

5. **查看训练产物并交付**

   脚本中 `project="runs/train"`、`name="exp"`，默认产物目录为：

   - `runs/train/exp/weights/best.pt`

   交付建议包含：

   - `best.pt`
   - 本次使用的 `dataset.yaml`
   - 训练指标说明（如 mAP、P/R、loss 曲线）

---

## 常见问题

| 现象 | 处理 |
|------|------|
| 训练时报找不到权重文件 | 检查 `app/train/yolo/weights/pretrained/yolov8s.pt` 是否在容器内存在 |
| 训练时报找不到数据集 | 检查 `dataset.yaml` 的 `path/train/val` 是否与容器内目录一致 |
| 训练结果没保存 | 确认 `runs/` 位于挂载卷内，避免容器退出后丢失 |
| 报 Python 包缺失 | 在镜像内补装依赖后重试，或将依赖 wheel 离线放入项目并安装 |
