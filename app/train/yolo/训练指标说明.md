# YOLO 训练指标说明（Ultralytics）

本文说明 `app/train/yolo/yolo_train.py` 训练过程中与训练结束后会产生哪些核心指标，以及如何查看、如何解读。

## 1. 会生成哪些指标文件

使用 `python app/train/yolo/yolo_train.py --config app/train/yolo/config.yaml` 训练后，Ultralytics 会在训练输出目录（通常是 `project/name`）生成：

- `results.csv`：每个 epoch 的指标明细（最核心）
- `results.png`：关键指标曲线图（loss、mAP、P、R 等）
- `confusion_matrix.png`：混淆矩阵（按类别看误检/漏检）
- `PR_curve.png`、`P_curve.png`、`R_curve.png`、`F1_curve.png`：检测性能曲线
- `weights/best.pt`：验证集指标最优权重
- `weights/last.pt`：最后一个 epoch 权重

> 注意：你的日志里显示 `save_dir` 类似  
> `/opt/workspace/models_app_framework/runs/detect/runs/yolo/helmet_exp4`  
> 指标文件就在该目录下。

---

## 2. 怎么查看指标

### 2.1 先定位输出目录

训练日志里会打印 `save_dir=...`，直接进入该目录查看：

```bash
cd /opt/workspace/models_app_framework/runs/detect/runs/yolo/helmet_exp4
ls
```

### 2.2 查看每轮详细指标

```bash
python - <<'PY'
import pandas as pd
df = pd.read_csv("results.csv")
print(df.tail(5))
PY
```

如果没装 pandas，也可用：

```bash
tail -n 5 results.csv
```

### 2.3 看可视化图

- 本地可直接打开 `results.png`
- 服务器环境可下载到本地查看，或在 Jupyter/VSCode Remote 中预览

## 3. 核心指标含义

以下以 YOLO 检测任务为主：

- `train/box_loss`：训练集框回归损失，越低越好
- `train/cls_loss`：训练集分类损失，越低越好
- `train/dfl_loss`：分布焦点损失（回归质量相关），越低越好
- `metrics/precision(B)`：精确率（预测为正中有多少是真的），越高越好
- `metrics/recall(B)`：召回率（真实目标中有多少被检出），越高越好
- `metrics/mAP50(B)`：IoU=0.5 时的 mAP，越高越好
- `metrics/mAP50-95(B)`：IoU=0.5~0.95 的平均 mAP（更严格），越高越好
- `val/box_loss`、`val/cls_loss`、`val/dfl_loss`：验证集损失，越低越好（用于判断泛化）

---

## 4. 参考值（经验区间）

不同任务差异很大（目标大小、类别数、标注质量、场景复杂度），以下仅作为经验参考：

- `mAP50`：
  - `< 0.40`：通常偏低（数据或训练策略需要明显优化）
  - `0.40 ~ 0.70`：中等可用（不少工业任务可先上线验证）
  - `> 0.70`：通常较好
- `mAP50-95`：
  - `< 0.20`：偏低
  - `0.20 ~ 0.45`：中等
  - `> 0.45`：较好（严格指标下表现不错）
- `Precision/Recall`：
  - 两者都 `> 0.70`：通常较稳定
  - Precision 高、Recall 低：漏检偏多（阈值/数据覆盖问题）
  - Recall 高、Precision 低：误检偏多（类别区分/负样本问题）

> 不建议只看单个指标。最佳实践是联合看 `mAP50-95 + PR 曲线 + confusion_matrix`。

---

## 5. 多少算“好”或“不好”（落地判断）

可按以下规则快速判断：

1. **先看趋势**
   - `train loss` 下降、`val loss` 也下降：正常收敛
   - `train loss` 下降但 `val loss` 上升：可能过拟合
2. **再看最终精度**
   - `mAP50-95` 是否达到你业务基线（例如安监检测常见目标可先以 0.30~0.40 为阶段目标）
3. **最后看业务代价**
   - 高漏检成本场景：优先提升 Recall
   - 高误报成本场景：优先提升 Precision

---

## 6. 指标不好时怎么改

- 数据层：
  - 补充难例、边界样本、遮挡样本
  - 清洗错误标注，统一标注规范
- 训练层：
  - 延长 `epochs`，适当调低 `lr0`
  - 调整增强（`mosaic/mixup/scale`）
  - 尝试更大模型（`yolov8s/m/l`）或更高分辨率 `imgsz`
- 推理层（上线前）：
  - 调 `conf` 与 `iou` 阈值，平衡 Precision/Recall

---

## 7. 建议的验收基线（可按项目微调）

可先设一个简单门槛：

- `mAP50 >= 0.65`
- `mAP50-95 >= 0.35`
- `Precision >= 0.70`
- `Recall >= 0.70`

若低于基线，优先检查数据集质量和类别定义，再做参数调优。
