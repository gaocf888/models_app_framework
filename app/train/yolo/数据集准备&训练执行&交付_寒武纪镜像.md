# 数据集准备 & 训练执行 & 交付（寒武纪现有镜像）

本文档用于在**寒武纪服务器**上，基于已安装训练栈（`ultralytics`、`torch`、`torchvision` 等）的现有镜像，执行本项目 `app/train/yolo/yolo_train.py` 的完整训练流程。

---

## 1. 前置条件

- 已有可用寒武纪服务器与可运行容器环境（docker/ctr 等，按你们平台规范）。
- 已有训练镜像（内含 Python + `ultralytics` + `torch` + `torchvision`）。
- 当前项目代码已同步到服务器某目录，例如：
  - `/data/projects/models_app_framework`
- 训练数据可访问（本地磁盘/NAS），并可挂载到容器内。

> 关键原则：`yolo_train.py` 使用项目根路径语义（会读取并改写 `data.yaml` 的 `path`），因此建议挂载**整个项目目录**到容器中，而不是仅挂载 `app/train/yolo` 子目录。

---

## 2. 目录与路径约定

建议在服务器保持如下结构（示例）：

```text
/data/projects/models_app_framework
└── app/train/yolo/
    ├── config.yaml
    ├── yolo_train.py
    └── datasets/
        ├── source/
        └── processed/
```

训练命令始终在**项目根目录**执行：

```bash
cd /data/projects/models_app_framework
PYTHONPATH=. python app/train/yolo/yolo_train.py --config app/train/yolo/config.yaml
```

---

## 3. 数据集准备

### 3.1 标准 YOLO 数据集

- 将数据集放到：
  - `app/train/yolo/datasets/processed/[你的数据集目录]`
- 确认 `dataset.yaml`（或同等 data yaml）中 `path` 为**项目根相对路径**，例如：

```yaml
path: "app/train/yolo/datasets/processed/handup_yolo_dataset"
train: images/train
val: images/val
names:
  0: class_a
  1: class_b
```

### 3.2 COCO 转 YOLO（如需）

- COCO 放到：
  - `app/train/yolo/datasets/source/[coco目录]`
- 按现有脚本配置转换路径后，在项目根执行：

```bash
PYTHONPATH=. python app/train/yolo/tools/convert_coco_to_yolo_run.py
```

---

## 4. 训练配置说明（config.yaml）

`app/train/yolo/config.yaml` 关键项：

- `pretrained_model`：可填本地权重路径，或 `yolov8n.pt` 这类模型名
- `data_yaml`：数据集 yaml 路径（必须存在）
- 其余顶层参数会透传给 Ultralytics `model.train(...)`
- 可选 `train_args`：额外训练参数（会 merge 到训练参数）

最小示例：

```yaml
pretrained_model: yolov8n.pt
data_yaml: app/train/yolo/datasets/processed/handup_yolo_dataset/dataset.yaml
project: app/train/yolo/runs
name: handup_exp01
epochs: 100
imgsz: 640
batch: 16
device: 0
workers: 8
```

> 说明：`yolo_train.py` 会基于项目根自动把 `data.yaml` 中 `path` 转为绝对路径临时文件供 Ultralytics 使用。

---

## 5. 容器启动与训练执行（寒武纪镜像）

以下为通用模板，请把 `<your_image>`、设备参数替换为你们实际值。

```bash
docker run --rm -it \
  -v /data/projects/models_app_framework:/workspace/models_app_framework \
  -w /workspace/models_app_framework \
  <your_image>:<tag> \
  bash -lc 'PYTHONPATH=. python app/train/yolo/yolo_train.py --config app/train/yolo/config.yaml'
```

### 5.1 设备透传

寒武纪设备透传参数（如 `--device`、`--privileged`、runtime 参数）请按你们平台规范追加。  
建议先在容器里验证一次设备可见性与 torch 可用性，再正式训练。

### 5.2 持久化建议

- `project` 输出目录应落在挂载卷内（如 `app/train/yolo/runs`），避免容器销毁后训练结果丢失。
- 如数据集不在项目仓库内，可额外挂载数据盘并在 `data_yaml.path` 中指向对应位置。

---

## 6. 训练后交付物

至少交付：

- `best.pt`（最佳权重）
- `dataset.yaml`（类别与数据集配置）
- 训练日志与关键指标（mAP、Precision、Recall、F1 曲线）

建议打包结构：

```text
your_model_name_deploy/
├── best.pt
├── dataset.yaml
└── README.txt
```

`README.txt` 建议包含：

- 模型名称
- 训练日期
- 输入尺寸（`imgsz`）
- 类别数与类别名
- 核心指标（mAP@0.5、mAP@0.5:0.95、Precision、Recall）
- 推理阈值建议

## 7. 常见问题排查

### 7.1 `data_yaml not found` / 路径错误

- 确认训练命令在项目根目录执行。
- 确认 `--config` 路径正确。
- 确认 `config.yaml` 的 `data_yaml` 指向容器内可见文件。

### 7.2 `dataset path` 解析异常

- 确认 `dataset.yaml` 的 `path` 是项目根相对路径（或容器内绝对路径）。
- 不要使用本地 Windows 路径（如 `F:\...`）直接放到服务器容器配置中。

### 7.3 容器内找不到 `ultralytics`

- 说明当前镜像不满足训练栈要求，需更换到你们已安装依赖的训练镜像。

### 7.4 结果丢失

- 检查 `project/name` 输出是否在挂载目录内。
- 检查容器是否使用了临时目录且未做卷挂载。
