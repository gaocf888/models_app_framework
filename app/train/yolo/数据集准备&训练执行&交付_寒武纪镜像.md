# 寒武纪镜像：YOLO 训练部署与执行（简版）

在寒武纪服务器上，用已有镜像（含 `torch`、`torchvision`、`ultralytics`）跑本项目 `app/train/yolo/yolo_train.py`。

---

## 你需要什么

- 服务器上有一份**完整项目目录**（不要只拷 `app/train/yolo`，脚本会按**项目根**解析 `data.yaml` 里的 `path`）。
- 镜像里若还没有 PyYAML，把 wheel **提前放进项目里**（见下文 `offline_pkgs`），`docker run` 时先本地安装再跑训练，**不必**把寒武纪镜像拿到外网。
- **不能上网时**：预训练权重 `.pt` 和数据集都要事先放进挂载目录，`config.yaml` 里 `pretrained_model` 写**本地路径**，不要写会从公网下载的名字。

---

## 怎么执行（按顺序做）

1. **把项目放到服务器**，例如：`/data/projects/models_app_framework`。

2. **放数据集**：例如放到 `app/train/yolo/datasets/processed/你的数据集/`，其中的 `dataset.yaml` 里 `path` 用**相对项目根**的路径，例如：

   ```yaml
   path: "app/train/yolo/datasets/processed/handup_yolo_dataset"
   train: images/train
   val: images/val
   names: { 0: a, 1: b }
   ```

3. **改 `app/train/yolo/config.yaml`**：至少设好 `pretrained_model`（本地 `.pt` 路径）、`data_yaml`（上一步的 yaml 路径）、`project`（建议仍在项目里，如 `app/train/yolo/runs`，方便挂载保存）。

4. **（仅离线 / 镜像未带 PyYAML）提前准备 wheel，放进项目**：

   - 在**任意能上网的环境**准备与寒武纪容器**一致**的 wheel：**Linux** + **CPU 架构**（`x86_64` / `aarch64`）+ **Python 小版本**（在寒武纪容器里执行 `python -V` 确认，例如 3.10）。
   - 在一台满足上述条件的 Linux 上（或公司同架构的 CI 镜像里）执行：

     ```bash
     mkdir -p offline_pkgs
     pip download pyyaml -d offline_pkgs
     ```

   - 将 `offline_pkgs` 目录放到本仓库：**`app/train/yolo/offline_pkgs/`**（与 `yolo_train.py` 同级下的子目录），随项目一起同步到寒武纪服务器。  
     若 `pip download` 拉下多个文件（依赖 wheel），**整个目录原样保留**，不要只拷一个文件。

5. **进容器：先装 PyYAML，再训练**（工作目录为项目根，`-w` 已保证；用 `python -m pip` 与训练共用同一解释器）：

   ```bash
   docker run --rm -it \
     -v /data/projects/models_app_framework:/workspace/models_app_framework \
     -w /workspace/models_app_framework \
     <your_image>:<tag> \
     bash -lc 'python -m pip install --no-index --find-links app/train/yolo/offline_pkgs pyyaml && PYTHONPATH=. python app/train/yolo/yolo_train.py --config app/train/yolo/config.yaml'
   ```

   若镜像里已有 PyYAML，上面 `pip install` 会快速完成或提示已满足要求，可保留同一命令，无需改脚本。

   寒武纪设备透传（`--device`、runtime 等）按你们平台加在 `docker run` 上。

   其它缺包（如 `tqdm`）也可把 wheel 放进同一 `offline_pkgs`，在同一 `bash -lc` 里追加：`python -m pip install --no-index --find-links app/train/yolo/offline_pkgs tqdm`（包名按需替换）。

---

## 交付

从 `project/name` 下取 `weights/best.pt`，连同 `dataset.yaml` 和训练指标说明一并交付即可。

---

## 常见问题（对照查）

| 现象 | 处理 |
|------|------|
| 找不到 `data_yaml` 或数据集 | 是否在项目根执行；`config.yaml` 里路径是否在容器内存在 |
| `No module named yaml` | 确认 `app/train/yolo/offline_pkgs` 已同步且 wheel 与容器 Python/架构匹配；或检查 `pip install` 是否在训练前执行成功 |
| 训练要联网下权重 | `pretrained_model` 改为已拷贝进挂载目录的本地 `.pt` |
| 容器删了结果没了 | `project` 必须落在挂载卷里（例如 `app/train/yolo/runs`） |

COCO 转 YOLO 仍用：`PYTHONPATH=. python app/train/yolo/tools/convert_coco_to_yolo_run.py`（需先按脚本改好路径；转换脚本可能依赖 `tqdm` 等，同样可离线安装）。
