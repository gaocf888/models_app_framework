## 重要说明
    当前版本训练整体代码(app/train/yolo)中ultralytics/是经过适配寒武纪修改后的ultralytics 8.3.75版本，
    训练脚本中 from ultralytics import YOLO时优先走当前路径下的ultralytics
    所以如果在nvidia卡中运行训练，需要把当前路径下的ultralytics文件夹删掉

## 数据集准备
    - 正常标准的yolo数据集，只需要放置到 datasets/processed/[数据集文件夹]路径下即可

    - COCO数据集使用方法：
        - COCO数据集放置到 datasets/source/[coco数据集] （包括_annotations.json和所有图片）
        - app/train/yolo/tools/convert_coco_to_yolo_run.py中修改 coco数据集 和 转换(yolo)后的路径(datasets/source/[coco数据集])
        - 项目根路径下执行数据集转换脚本：PYTHONPATH=. python app/train/yolo/tools/convert_coco_to_yolo_run.py


## YOLO训练

    - app/train/yolo/config.yaml修改训练参数，特别是 data_yaml （上述数据集的配置文件，一般是在 datasets/processed/[数据集文件夹]中）

    - app/train/yolo/datasets/processed/[数据集文件夹]中修改数据集配置文件（修改path），如下（目前需要配置为基于项目根路径的 相对路径）
    path: "app/train/yolo/datasets/processed/handup_yolo_dataset"

    - 执行训练脚本： PYTHONPATH=. python app/train/yolo/yolo_train.py --config app/train/yolo/config.yaml


## 训练后交付

    - 模型权重文件：best.pt

    - 类别标签文件：dataset.yaml

    - 关键训练指标说明：
        - mAP@0.5： [<0.5-较差,>0.5-一般,>0.7-良好,>0.85-优秀,>0.95极好]    --  模型在IoU阈值为0.5时的平均精度 -- 提升：增加训练轮次、使用更大的模型、调整图像大小(调大)、数据增强(augment=True)、调整学习率

        - mAP@0.5:0.95： [在 IoU=0.5 到 0.95 之间取10个阈值，计算平均值]  --  模型在IoU阈值为0.5时的平均精度(更严格的综合指标)

        - Precision (精确率)：

        - Recall (召回率)：

        - F1-Confidence Curve(分数): 

    - 模型输入要求说明：

        - 输入尺寸 (imgsz)：训练时用的图片大小，如 640x640。同事部署时需将图片缩放到此尺寸
        
        - 预处理方式：通常包括归一化（除以255）、通道变换（HWC转CHW）等。如果是标准YOLO，这些一般是固定的，但最好确认一下


## 建议交付格式

    your_model_name_deploy/
├── best.pt                 # 核心模型文件
├── dataset.yaml            # 类别名称文件
└── README.txt              # 简单的说明文档
